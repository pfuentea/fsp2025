Manual paso a paso para instalar la app reusable core_auth en otro proyecto Django

Este manual explica cómo tomar la app core_auth que ya tienes lista y conectarla a un proyecto Django nuevo o existente.

Resultado esperado: al terminar, el proyecto tendrá registro, login, logout, perfil y edición de perfil funcionando.
Paso 1

Copiar la carpeta de la app

Copia la carpeta core_auth completa dentro de tu proyecto Django.

La estructura debería quedar parecida a esto:

mi_proyecto/
├── manage.py
├── mi_proyecto/
│   ├── settings.py
│   ├── urls.py
│   └── ...
├── core_auth/
│   ├── __init__.py
│   ├── admin.py
│   ├── apps.py
│   ├── forms.py
│   ├── models.py
│   ├── signals.py
│   ├── urls.py
│   ├── views.py
│   └── templates/
│       └── core_auth/
│           ├── login.html
│           ├── register.html
│           ├── profile.html
│           └── profile_edit.html
Si tu proyecto usa una carpeta apps/, también puedes dejarla así: apps/core_auth/. En ese caso debes ajustar el nombre de la app en apps.py y en INSTALLED_APPS.
Paso 2

Registrar la app en settings.py

Abre settings.py y agrega la app dentro de INSTALLED_APPS:

INSTALLED_APPS = [
    # apps de Django
    "django.contrib.admin",
    "django.contrib.auth",
    "django.contrib.contenttypes",
    "django.contrib.sessions",
    "django.contrib.messages",
    "django.contrib.staticfiles",

    # tu app reusable
    "core_auth.apps.CoreAuthConfig",
]
Es importante usar CoreAuthConfig para que el método ready() cargue los signals automáticamente.
Paso 3

Configurar las redirecciones de autenticación

En settings.py agrega estas configuraciones:

LOGIN_URL = "core_auth:login"
LOGIN_REDIRECT_URL = "core_auth:profile"
LOGOUT_REDIRECT_URL = "core_auth:login"

Con esto:

  • si una vista requiere login, Django enviará al usuario a la pantalla de login
  • después de iniciar sesión, irá al perfil
  • después de cerrar sesión, volverá al login
Paso 4

Configurar templates

Revisa que tu proyecto tenga habilitado el uso de templates en settings.py.

TEMPLATES = [
    {
        "BACKEND": "django.template.backends.django.DjangoTemplates",
        "DIRS": [BASE_DIR / "templates"],
        "APP_DIRS": True,
        "OPTIONS": {
            "context_processors": [
                "django.template.context_processors.request",
                "django.contrib.auth.context_processors.auth",
                "django.contrib.messages.context_processors.messages",
            ],
        },
    },
]

La opción clave aquí es:

"APP_DIRS": True

Eso permite que Django encuentre los templates dentro de la carpeta templates/ de la app core_auth.

Paso 5

Configurar mensajes

La app usa messages para mostrar avisos como “cuenta creada correctamente” o “perfil actualizado”.

Normalmente ya viene listo en Django, pero revisa esto en settings.py:

INSTALLED_APPS = [
    ...
    "django.contrib.messages",
]
MIDDLEWARE = [
    ...
    "django.contrib.sessions.middleware.SessionMiddleware",
    "django.contrib.auth.middleware.AuthenticationMiddleware",
    "django.contrib.messages.middleware.MessageMiddleware",
    ...
]
Paso 6

Configurar media para la foto de perfil

Como el modelo Profile usa un campo ImageField, debes configurar archivos media.

En settings.py agrega:

MEDIA_URL = "/media/"
MEDIA_ROOT = BASE_DIR / "media"

Y en el urls.py principal del proyecto agrega:

from django.contrib import admin
from django.urls import path, include
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    path("admin/", admin.site.urls),
    path("auth/", include("core_auth.urls")),
]

if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
Esto permite que en desarrollo puedas ver las imágenes subidas por los usuarios.
Paso 7

Incluir las URLs de la app

En el archivo urls.py principal del proyecto debes importar las rutas de core_auth.

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path("admin/", admin.site.urls),
    path("auth/", include("core_auth.urls")),
]

Con esto tendrás disponibles estas rutas:

Ruta Funcionalidad
/auth/login/Iniciar sesión
/auth/logout/Cerrar sesión
/auth/register/Registro
/auth/profile/Ver perfil
/auth/profile/edit/Editar perfil
Paso 8

Verificar el template base del proyecto

Los templates de core_auth extienden de un archivo llamado base.html. Por eso, el proyecto donde la instales debe tener un template base.

Ejemplo mínimo de templates/base.html:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Proyecto</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
    <nav class="navbar navbar-expand-lg navbar-dark bg-dark">
        <div class="container">
            <a class="navbar-brand" href="/">Mi Proyecto</a>

            <div class="ms-auto">
                {% if user.is_authenticated %}
                    <a href="{% url 'core_auth:profile' %}" class="btn btn-outline-light btn-sm me-2">Perfil</a>
                    <form method="post" action="{% url 'core_auth:logout' %}" class="d-inline">
                        {% csrf_token %}
                        <button type="submit" class="btn btn-danger btn-sm">Salir</button>
                    </form>
                {% else %}
                    <a href="{% url 'core_auth:login' %}" class="btn btn-outline-light btn-sm me-2">Login</a>
                    <a href="{% url 'core_auth:register' %}" class="btn btn-success btn-sm">Registro</a>
                {% endif %}
            </div>
        </div>
    </nav>

    {% block content %}{% endblock %}
</body>
</html>
Si ya tienes un base.html propio, no necesitas reemplazarlo. Solo asegúrate de que tenga el bloque {% block content %}{% endblock %}.
Paso 9

Instalar Pillow

Como el modelo usa ImageField, Django necesita la librería Pillow.

pip install Pillow

Si usas un archivo de dependencias, agrégala también a requirements.txt:

Pillow
Paso 10

Crear y aplicar migraciones

Ahora debes generar la tabla del modelo Profile en la base de datos.

python manage.py makemigrations core_auth
python manage.py migrate

Opcionalmente crea un superusuario para probar también desde el admin:

python manage.py createsuperuser
Paso 11

Levantar el servidor y probar el flujo

python manage.py runserver

Prueba este flujo:

  1. entrar a /auth/register/
  2. crear un usuario
  3. verificar que redirige a /auth/profile/
  4. editar el perfil en /auth/profile/edit/
  5. cerrar sesión
  6. iniciar sesión otra vez en /auth/login/
Si todo sale bien, la app reusable quedó instalada correctamente.
Paso 12

Cómo usarla en vistas protegidas

Una vez instalada, puedes proteger otras vistas del proyecto usando @login_required.

from django.contrib.auth.decorators import login_required
from django.shortcuts import render

@login_required
def dashboard(request):
    return render(request, "dashboard.html")

Si el usuario no ha iniciado sesión, Django lo enviará automáticamente a:

/auth/login/
Paso 13

Cómo extender la información del usuario después

Si más adelante quieres agregar nuevos campos, no necesitas modificar el modelo User. Solo agregas nuevos campos a Profile.

class Profile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE, related_name="profile")
    telefono = models.CharField(max_length=30, blank=True, null=True)
    rut = models.CharField(max_length=20, blank=True, null=True)
    direccion = models.CharField(max_length=255, blank=True, null=True)
    empresa = models.CharField(max_length=150, blank=True, null=True)

Luego ejecutas nuevamente:

python manage.py makemigrations
python manage.py migrate

Y agregas esos campos al formulario ProfileUpdateForm.

Paso 15

Errores frecuentes

1. TemplateDoesNotExist

Verifica que:

  • APP_DIRS = True
  • la carpeta sea templates/core_auth/
  • el nombre de los archivos coincida exactamente

2. No module named PIL

Debes instalar Pillow:

pip install Pillow

3. NoReverseMatch con core_auth

Verifica que en core_auth/urls.py exista:

app_name = "core_auth"

4. No aparece la foto

Revisa:

  • MEDIA_URL y MEDIA_ROOT
  • las rutas estáticas para media en el urls.py principal
  • que estés en modo DEBUG = True durante desarrollo

5. El perfil no existe

Revisa que la app esté cargando CoreAuthConfig y no solo core_auth en INSTALLED_APPS.

Conclusión

Con esta instalación, core_auth queda desacoplada del resto del proyecto y lista para reutilizarse en nuevas aplicaciones Django. La ventaja principal es que concentras la autenticación en una app propia, manteniendo flexibilidad para extender datos de usuario mediante Profile.
